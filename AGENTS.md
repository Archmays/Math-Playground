# AGENTS.md

本文件是数学游乐场项目对 AI 编码助手的长期规则。除非用户明确覆盖，所有后续修改都应遵守这些规则。

## 项目定位

- 项目中文名：数学游乐场
- 项目英文名：Math Playground
- package name：`math-playground`
- 这是原创教育项目，不是 Polypad 克隆。
- 不复制或近似复刻 Polypad 的品牌、UI、图标、素材、源码或专有交互细节。
- 可以借鉴“虚拟数学教具 + 可拖拽画布 + 教师任务模板”的通用产品理念。
- 第一版面向小学到初中低年级，不做大型平台。

## 工作方式

- 先想清楚再编码。说明关键假设；需求有多种解释时先指出。
- 对多步骤任务给出简短计划，并写明如何验证。
- 优先做最小可用实现，不添加未请求的功能。
- 变更要外科手术式：只改和当前请求直接相关的文件。
- 匹配现有代码风格，不顺手重构无关代码。
- 如果发现无关问题，记录或告知用户，不直接改。

## 技术约束

- 使用 Vite、React、TypeScript。
- 使用 SVG 作为主要 2D 画布渲染方式。
- 样式使用普通 CSS 或 CSS Modules，暂不引入大型 UI 框架。
- 核心逻辑使用 Vitest 测试。
- 暂不引入 Redux；优先使用 React reducer/context 管理状态。
- 不引入后端、账号、云同步或支付，除非用户明确要求。

## 代码结构

- `src/core`：核心数据模型、序列化、纯函数逻辑。
- `src/canvas`：SVG 画布、坐标系、选择、拖拽、缩放等画布能力。
- `src/manipulatives`：数学教具类型、默认属性、渲染入口。
- `src/features`：面向用户流程的功能模块，例如任务模板、工作区。
- `src/components`：可复用 UI 组件。
- `src/utils`：通用工具，不包含业务状态。

## 数据模型

- 所有教具对象必须能序列化为 JSON。
- 新增教具前先定义稳定的 `kind`、基础几何属性和 `data` 结构。
- 避免把 React 组件、函数、DOM 节点或不可 JSON 化对象放进核心模型。
- 保存格式需要包含 `schemaVersion`，为未来迁移预留空间。

## 测试与验收

- 新增核心逻辑时优先先写测试，再实现。
- 每次完成任务前至少运行相关测试；涉及构建或类型时运行：
  - `npm run build`
  - `npm run typecheck`
  - `npm run test`
- 不用“应该能过”替代实际命令输出。
